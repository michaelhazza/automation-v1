Parallel-but-distinct: the shell (header, filter tabs, card grid) mirrors the Workflows library shape. The visual distinctions are engine-coloured avatars, the engine-filter chip row, and the connection-readiness signal — all Automations-specific.
Engine picker on empty state: five big buttons for Make/n8n/GHL/Zapier/Webhook make the next action obvious. Agency owners don't need to learn what "Automation" means before picking one — the engines are already familiar vocabulary.
Connection readiness on every card: tells the user at-a-glance which Automations are actually callable. This signal is the same one shown inside the picker drawer (mock 6) — visual language is consistent.
