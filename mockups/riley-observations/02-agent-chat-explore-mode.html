Chip placement decision: top-right, always visible, pill with lock-icon for Explore vs play-icon for Execute. A single click opens a confirm dialog (not shown) before switching — avoids accidental toggles mid-action.
Naming critique hook: "Explore" = safe discovery; "Execute" = act for real. Alt considered + rejected: "Preview / Run" (collides with existing Workflow run terminology), "Dry-run / Live" (too engineer-coded for agency owners).
Mid-conversation switch: rendered as a system message so the audit trail is readable inline. Matches the pattern in spec §6.8 "Mode changes mid-conversation are logged as system messages."
Why Explore earns its keep: the amber "Waiting for approval" block is the user-visible payoff. Every side-effecting action lands here in Explore Mode — this is the screen that makes the safety claim concrete.
