How to use this review surface

  1. Walk top-to-bottom. Each card names its open question; click through, read the inline annotations, form a verdict.
  2. If a mock resolves the question → note it in the spec as decided (copy the chosen copy verbatim if applicable).
  3. If a mock raises a new question → add it to tasks/todo.md under the existing "Deferred from spec-reviewer review — riley-observations-dev-spec" section.
  4. When all ten verdicts land, the spec is ready for architect decomposition.
Mockups are static HTML. Use File → Open in a browser, or serve the folder locally (e.g. python -m http.server from mockups/riley-observations/).
