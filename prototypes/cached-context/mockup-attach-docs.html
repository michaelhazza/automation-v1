When the suggestion fires

The suggestion above is a post-save prompt, not a mid-flow interruption. It appears only when all of these are true:

  • The user attached 2 or more documents.
  • The exact same doc set (same IDs, any order) is already attached to ≥ 1 other task/agent/scheduled-task. Detection uses the prefix_hash primitive the spec already computes — no fuzzy matching, no similarity thresholds.
  • The user hasn't dismissed this exact doc-set suggestion before (permanent dismissal; won't re-ask).
  • The user doesn't already have a named bundle with this exact doc set (future enhancement: prompt changes to "use your existing bundle" instead).

Accepting opens a tiny "name your bundle" input. Dismissing hides the suggestion forever for this doc set. Bundles are an opt-in efficiency affordance — nothing in the attach picker references them until the user has at least one.
