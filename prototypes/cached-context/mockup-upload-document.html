How this works

Empty state (before any file is picked): the file list is replaced by a dashed drop zone with a cloud icon and "Drop files here, or click to browse · Accepts .md, .pdf, .txt, .docx". Dragging files onto the modal anywhere triggers upload — the list is a drop target too.

Bundle checkbox: off by default. Visible whenever 2+ files are staged (grayed out on single-file uploads — no bundle for one document). When ticked, the bundle name input appears and the confirmation line changes to name the bundle.

Reusable across the site: same dialog opens from the attach picker (auto-attaches to the current task/agent/scheduled-task after upload) and from the standalone Knowledge › Documents page (no auto-attach, just adds to the library). The auto-attach line only appears in the task-scoped context.
