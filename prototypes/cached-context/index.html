Cached-context infrastructure — attachment mockups

Three mockups centered on the primary user task: attach documents to automations. Documents are the first-class noun. "Bundles" appear only as a soft opt-in suggestion after the user has attached the same doc set to two places — never as a prerequisite.

Spec · docs/cached-context-infrastructure-spec.md UI rules · docs/frontend-design-principles.md v2.1 scope · 4 mockups
The concept. Users attach individual documents. When a user attaches the same doc set to a second task, the UI offers "save these as a bundle for easy reuse?" — soft, dismissible, non-blocking. Exact-match detection via the spec's existing prefix_hash, no fuzzy logic. The backend handles everything as a `document_bundles` row (unnamed until the user opts in to name it); users never see the "unnamed" variant.

Primary user journey

Attach documents → (optionally save as a bundle when it makes sense) → run.

Attach reference documents

The hero of this feature. Documents show up as inline chips. The picker has a single search field across documents AND bundles — two labelled sections, one search input. Reference material too big — what now?

Safety-critical screen. Headline is "the reference documents attached to this task are too big" regardless of whether they came from a bundle or were attached individually — doc-framed, not bundle-framed. The over-limit bar, top-5 contributors, and 4 actions (Remove/trim · Upgrade · Split · Stop) are all the user needs to decide what to do.

What changed vs the previous set

Before — capability-first UX

  • User attaches a named group as the primary unit
  • Must create the named group before attaching anything
  • The internal grouping concept is surfaced as a first-class noun everywhere
  • A full management page is a v1 requirement
  • Per-tier utilization badge on the attachment row

After — doc-first UX

  • User attaches individual documents directly
  • No concept the user has to learn before attaching
  • "Bundle" only appears as an opt-in suggestion
  • Bundle detail page is opt-in · appears only after user saves one
  • No health badge in the primary flow · moved to bundle detail "Advanced"
