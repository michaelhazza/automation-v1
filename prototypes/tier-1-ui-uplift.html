Design decisions to confirm: Prompt chips (white/slate) vs system chips (indigo tinted) are visually distinct. Chips only appear on the most recent assistant message, disappear when the user sends a new message. Max 4 chips. Design decisions to confirm: Card width: 340px max-width, aligned with the assistant message column (not full-width). "Dismiss" collapses the card to a minimal 1-line stub — the run stays blocked until expiry or user reconnects via "Connect" in the stub. "Try again" on the expired state creates a new run, not a resume of the old one. OAuth popup vs. same-tab redirect? Lean: popup, with postMessage back on success. Should the TTL countdown be live (ticking) or static on render? Multiple integration blocks in one run: each generates its own card in sequence — no batching.
