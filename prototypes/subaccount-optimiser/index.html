home-dashboard-subaccount-context.html
What changed from v3
  • Strip on the sub-account detail page is dropped. That page is a 13-tab settings shell — wrong neighborhood for suggestions. Pushing the tabs down broke the page's purpose.
  • Per-sub-account suggestions now live on the Home dashboard, just scoped. The Home page is the same `/` route in both contexts; only the sidebar differs. Make the suggestions section scope-aware and you cover both audiences without inventing a new page or hijacking a settings page.
  • Open architectural question this exposes. The Home dashboard already has a pre-existing tension — "Clients Needing Attention" doesn't make sense in sub-account context, "Active Agents" is org-scoped today. This isn't created by suggestions; it's just visible because we're adding context-aware behavior. Worth flagging in the spec.
