What changed from v1

  • No KPI strip. The "12 things to look at" line in the header is the only count on the page. Operators don't need to know the breakdown by severity to start triaging.
  • No category filter chips. One sort dropdown ("Most important first" / "By client" / "Newest first"). Categories are an internal concept; the operator doesn't pick from them.
  • One action per row. "Help me fix this →" only. No acknowledge button, no icon column. The × is a soft "not now" — the page doesn't pretend each item is a workflow with three states.
  • Plain language across all 8 categories. No agent.over_budget, no llm.cache_poor_reuse — those exist only in the database.
  • One narrow column. The page is ~820px wide. Long-line scanning beats wide-table scanning for a triage list.
