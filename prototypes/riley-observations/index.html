Riley Observations — Mockups

Ten minimal mockups covering the user-facing changes in the spec. Each one shows the primary user task for its surface — nothing more.

Design rules applied
Start from the user's task, not the data model · default to hidden for dashboards / diagnostics / ID exposures · one primary action per screen · inline state beats dashboards · complexity budget enforced per docs/frontend-design-principles.md.

Removed from v1 (moved to admin, backlog, or just cut)
  • Heartbeat gate activity chart and skip-rate KPI — admin observability page, not per-agent sidebar.
  • Run Detail page exposing request JSON, response JSON, tracing events, HTTP status — admin concerns.
  • Error-code reference grid — docs, not UI.
  • Engine filter chips and scope filter tabs on the libraries — default views handle this.
  • Trust-receipts list on the promote-to-Execute prompt — the counter ("5 successful runs") is the receipt.
  • Workflow-Studio right inspector with composition metrics — inline step count appears where needed.
  • Mode-switch mid-conversation system-message demo — logged in the run log, not inline-reinforced.
  • Per-row step-count chips on Workflows cards — data-model leak.
  • Design-decision annotation dashboards on each mock — replaced with a single explanatory line.
Part 5 (context-assembly telemetry) and Part 6 (decomposition rule) have no v1 UI and aren't mocked. Part 4's admin observability also has no v1 UI; only the per-agent settings surface (mock 10) is shown.
