Decision surfaced · UX

How visible should intermediate steps be?

Default this mockup uses: hidden by default, one progress chip ("Working across 3 teams…") with an expandable log. No per-agent trace in the user-facing chat.

Alternative: reveal each hop ("Checking competitor pricing…", "Drafting body…"). More trust-building, but noisier — and the user already has the delegation-graph tab for forensics.

Decision surfaced · Latency

Progressive reveal, or a single arrival?

Default this mockup uses: indeterminate progress bar + one status chip rotating through high-level phases every 3s. When the answer is ready, it appears as one CEO-voice message.

Alternative: stream partial drafts as the worker subtrees complete. Adds UI complexity; defers the "one agent, one answer" promise.

Decision surfaced · Failure

What does a failed sub-task look like?

Default this mockup uses: the manager handles it; the user never sees the failure. The delegation graph still records the rejection with full reason — Phase 4 enforcement guarantees the audit trail.

Only if the top-level Brief fails does the user see a degraded response, with a "View what happened" link to the graph view.
