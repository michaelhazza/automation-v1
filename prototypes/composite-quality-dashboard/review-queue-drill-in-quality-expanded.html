Designer notes — review-queue-drill-in-quality-expanded.html
Extends: client/src/pages/ReviewQueuePage.tsx (queue dimmed behind) + DrillInDialog Level 3 view.
This is Level 3 of the drill-in pattern (brief §3.6). The breadcrumb "All dimensions / Quality" lets the operator navigate back to the Level 2 view. The 30-day sparkline has band-coloured background bands (green/amber/red zones) to give visual context to the score's position. Input metrics are shown as stat cards with progress bars. Raw verdicts list up to 5 rows; excess items link to the existing run log ("View all in run log"). Corrections list also links to existing pages.
"Insight" box surfaces a pattern (two skills dominate failures) as a deterministic observation — not LLM generated, consistent with Phase 1 rule-based callouts (brief §3.3). Links to amendment review for direct action. The dialog does NOT replicate the run log or amendment page; it points into them (§3.6: "The dashboard does not duplicate any existing page; it points into them.").
Principles: (1) workflow-anchored: operator is still in the review queue flow; (2) default-to-hidden: Level 3 is 2 clicks deep; (3) one primary action: go to amendment review or close; (4) inline state: sub-score + sparkline + verdict list communicate the full picture; (5) re-check: non-technical operator can read "72% pass rate, 38% correction rate" and understand why Quality is amber.
