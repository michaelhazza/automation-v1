Designer notes — system-quality-org-rollup.html
Extends: client/src/pages/SystemPnlPage.tsx — page layout (min-h-screen bg-slate-50, max-w-7xl, px-6 py-6), header pattern (h1 + description + right-aligned actions), KPI card grid (grid-cols-4), view toggle buttons (inline-flex border bg-white), table structure, footer note row. Reuses .data-table, PnlKpiCard visual shape, PnlGroupingTabs button pattern.
Staff admin view per §3.5 — relaxed design budget (4 KPI tiles, charts allowed, more table columns). Operates under the "admin-only views" exception from frontend-design-principles.md. Not customer-facing, gated behind /system/* auth. KPI tiles count: 4 (total orgs, red count, amber count, green count) — all actionable for staff triage.
Default sort: worst composite first (ascending). This surfaces the orgs needing immediate attention at the top. Sortable by: composite, trend, pending amendments, last activity. Attention callout column is truncated at ~220px with title attribute for full text on hover. "Drill in" links to org drilldown page.
Principles: (1) workflow-anchored: staff opens this page to identify which orgs need support; (2) default-to-hidden: N/A for staff view (relaxed budget); (3) one primary action: drill into a specific org; (4) inline state: band-coloured score cells + trend arrows tell the story at a glance; (5) re-check: Synthetos staff, not consumer operators (higher complexity tolerance).
