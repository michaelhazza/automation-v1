Designer notes — review-queue-drill-in-dimensions.html
Extends: client/src/pages/ReviewQueuePage.tsx (queue behind) + new DrillInDialog component.
Backdrop dimming: the queue is rendered with filter:brightness(0.45) and pointer-events:none to simulate the modal overlay. In production this would be a React portal with backdrop. Dims the entire page including sidebar and queue list.
Dialog structure: composite summary bar (amber, full-width at top) then four collapsible dimension rows. Quality is expanded (default for this mockup) to show inputs as metric rows and 3 recent signal verdicts. Three-dot chevron indicates collapsed/expanded state. Sparklines are 80x24px SVG inline polylines (30-day range, daily points). Sub-scores: Quality 71 (amber), Stability 63 (red), Cost 88 (green), Capacity 46 (red). Weighted average: (71+63+88+46)/4=67 shown in the formula line.
"See all failed verdicts and corrections" links to review-queue-drill-in-quality-expanded.html (Level 3). Each verdict row has a "View run ↗" link pointing to the existing run detail page — no duplication per §3.6.
Principles: (1) workflow-anchored: operator is still on the review queue page; (2) default-to-hidden: dialog is 1-click deep, not on the default view; (3) one primary action: review the expanded dimension or close and return to the queue; (4) inline state: sparklines + sub-scores tell the dimension story; (5) re-check: non-technical operator can read Quality 71, Stability 63, Cost 88, Capacity 46 and identify the problem dimensions.
