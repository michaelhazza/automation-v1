Workflow step types

The four A's

Every step in a workflow is one of four types. Memorise these four and you have the whole authoring vocabulary — branching and parallel work are properties of these steps, not separate concepts.

Agent
Hand it to an employee

Delegate the step to an agent with a free-form instruction. The agent picks the skills, integrations, and sub-tasks it needs at runtime.

When to use
Anything that needs judgement, writing, classification, or a sequence the agent should figure out.
Example
"Copywriter drafts a personalised follow-up email using the lead's history."
Output you can branch on
Anything the agent returns — classification, tier, status, count.
Action
Do one exact thing

A specific deterministic operation — call one skill (Slack message, GHL email, HubSpot update) or trigger one external automation (n8n, Make, Zapier, GHL flow).

When to use
You know exactly what should happen. No agent judgement needed. Faster, cheaper, predictable.
Example
"Add the contact to the GHL nurture-list tag."
Output you can branch on
Success / failure, return code, or a field from the response.
Ask
Get input from a human

Pause the workflow and present a form. Whoever's running the task fills it in; the values are available to subsequent steps.

When to use
You need information that only a human has — venue, target audience, choice between options.
Example
"Confirm event venue, capacity, and ticket price before drafting copy."
Output you can branch on
The form-field values the human entered.
Approval
Yes / no gate

Pause for a human's explicit yes / no. Optionally route back to a previous step on rejection (the only place a workflow can loop).

When to use
Before something irreversible — sending a campaign, publishing, charging a card, posting publicly.
Example
"Marketing reviews the email + recipient list before send."
Output you can branch on
approve · reject (and on reject: route back to a previous step).
