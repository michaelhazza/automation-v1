Workflows · UI prototypes

Brainstorming the Phase-2 UI for the workflows feature. Backend (engine, schema, routes) is already built — these mocks explore the operator surface, the Studio authoring surface, and the open-task surface.

Current direction (the four A's)

04 · Four A's — per-step-type inspector showcase
Foundation
Single page with four state-switch buttons at the top (Agent · Action · Ask · Approval) — click each to see that step type selected in a real workflow with its inspector showing fields specific to that type. Same canvas across all four states; only the selected step + inspector content change. Approval state demonstrates the new humans-only approver picker (Specific people / A team / Task requester / Org admin) with quorum.
05 · Studio — route editor
Recommended
Canvas-first authoring (chat as a docked pill, slide-out inspector, hover-to-insert "+", floating bottom action bar). Step cards use the four A's vocabulary. Branching as an inline "Branch on X" chip below an Agent step. Approval-on-reject as a dashed back-arrow to a previous step. Selected step is the Marketing review approval — inspector shows the new humans-only approver picker with quorum + loop-back-on-reject configuration.
06 · Open task view
Operator surface
Combines brief-endtoend.html's layout (full-width, chat + org-chart Live view + activity with linked-entity chips) with the task surface. Same view for ad-hoc and workflow-fired tasks — "Brief" is retired as a separate noun. Right panel has four distinct tabs: Live (who is working right now — org chart of agents) · Flow (the planned shape — step DAG or delegation tree) · Activity (full chronological history — view/edit chips on linked CRM records, docs, rules) · Files (artifacts produced). Renders the "Acme renewal proposal" brief-style task with COO orchestrating Sales / Research / Operations teams.
