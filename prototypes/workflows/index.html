Workflows · UI prototypes

Brainstorming the Phase-2 UI for the workflows feature. Backend (engine, schema, routes, step types including invoke_automation for n8n / Make / Zapier / GHL) is already built — these mocks explore the operator surface.

01 · Chat-driven Studio with live preview
Rejected as too busy
Three-pane workspace: chat / canvas / inspector all permanently visible. Per-step cost and side-effect badges. Multi-metric validation summary. Felt like Zapier with a chat overlay — violated frontend principles (≥2 primary actions, KPI chrome by default).
02 · Canvas-first with chat as a power tool
Recommended
Canvas is the whole screen. Direct manipulation handles 80% of edits — click to edit, hover-to-insert "+" between steps. Chat is a docked pill, opens on demand for big restructures. Step cards show only icon, name, action verb. Cost rolls up to one bottom-bar number. Only irreversible steps wear a flag. Inspector is a slide-out, not a permanent column.
