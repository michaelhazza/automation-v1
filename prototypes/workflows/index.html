Workflows · UI prototypes

Brainstorming the Phase-2 UI for the workflows feature. Backend (engine, schema, routes) is already built — these mocks explore the operator surface, the Studio authoring surface, and the open-task surface.

Current direction (the four A's)

04 · The four A's (step type showcase)
Foundation
Single page establishing the visual language. Workflow steps collapsed from eight engine types to four user-visible categories: Agent · Action · Ask · Approval. Branching is a property of the producing step (not its own type), parallel allowed via fan-out / fan-in, loops only on Approval-reject. Sample workflow at the bottom shows all four working together.
05 · Studio — route editor
Recommended
Replaces mocks 02 / 03. Same canvas-first layout (chat as a docked pill, slide-out inspector, inline insert "+", floating bottom action bar) but step cards now use the four A's vocabulary. Branching shown as an inline "Branch on X" chip below an Agent step. Approval-on-reject visualised as a dashed back-arrow to a previous step. The selected step (Marketing review) demonstrates the loop-back configuration in the inspector.
06 · Open task view
Operator surface
Models off the existing AgentChatPage layout (chat left, contextual right panel). The user-facing surface for any task in flight, whether it came from Ask Anything one-off or a scheduled workflow. Renders the lead re-engagement workflow from mock 05 mid-run, paused at the Marketing review approval — drafts done, 1 of 2 approvers signed off, irreversible Mailchimp send waiting. Right panel has tabs for Flow / Activity / Files; default Flow tab shows the route through agents and gates with done / current / pending state per leg.
