Key design decisions in this round
· "Skill improvements" is a new section below the Needs Review tab content, not a third tab. This keeps the Inbox tab count (Briefs/Needs Review) unchanged and avoids splitting the operator's attention.
· Jargon ("provenance chain", "lineage", "resolver version", "composition snapshot", "blast radius", "occurrence count") lives exclusively inside "Show technical detail" / "Show advanced details" expanders, collapsed by default on every screen.
· Reject reasons are 3 plain-English buttons ("Not the right fix" / "Don't want this here" / "Unsafe: don't suggest again"), not enum strings. The confirm button is disabled until one is selected.
· The cap-reached state (20 active improvements) shows as a small amber notice inside the expanded skill row, not a separate page. The pause toggle replaces the governance freeze screen entirely.
· The skills table drilldown stays inline (expandable row) rather than opening a new detail page, matching the brief constraint "NOT as a new detail page."
· No em-dashes anywhere. No new sidebar nav items. Sidebar in all 4 screens matches what exists in the codebase today.
