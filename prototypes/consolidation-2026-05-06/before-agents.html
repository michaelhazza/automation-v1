Current state, Agents (3 separate pages, same data, different scopes)

The same agents list exists in three places depending on who you are and what URL you visit. AdminAgentsPage has tabs (List / Org Execution / Team Templates) plus an "Install from System Library" modal. AgentsPage is the subaccount-scoped card grid with heartbeat status. SubaccountAgentsPage is a third variant scoped to a specific client. All three make the same /api/agents call.

Problems with the current state
  • • Three separate pages serve essentially the same data with different layouts, forcing the operator to know which URL to visit based on role.
  • • No inline last-run status on any list view. Operator must click into each agent to see when it last ran or if it failed.
  • • Card grid (AgentsPage) has no actions. Table (AdminAgentsPage) has 6 columns. Neither has a consistent pattern.
  • • "Team Templates" and "Org Execution" tabs in AdminAgentsPage embed entire separate pages (SubaccountBlueprintsPage, OrgAgentConfigsPage) inside a tab -- creating visual mismatch.
  • • No quick path from the agents list to running an agent or seeing recent runs inline.
