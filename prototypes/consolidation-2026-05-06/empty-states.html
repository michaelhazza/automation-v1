Empty States

Canonical empty-state patterns for each major list/collection in Automation OS. Each pattern has exactly one primary action. Builder reference, not a real page.

Every empty state has exactly one primary action. Not two, not a tour, not tips.
The action should create the first item of that type or navigate to the right place to do so.
Short headline: "No [thing] yet." Short body: one sentence about what [thing] does. CTA: the logical first step.
No empty-state chart, no placeholder data, no "here's what you could do" lists.
