Pattern Reference

Lifecycle Error Patterns

How errors and blocked states surface in the UI. Each error state offers exactly one primary recovery action. Builder reference, not a real page.

Every error state has one primary recovery action. Show the action inline, not in a separate settings page.
Explain what failed in plain language, not in technical terms. "Gmail credential expired" not "OAuth token invalid".
Blocked states are amber (recoverable) or red (failed). Distinguish them visually.
Progress states (migration, syncing) are indigo, never red -- they are expected, not errors.
