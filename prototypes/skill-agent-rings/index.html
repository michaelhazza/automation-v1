Design notes

  • Two user classes, different rules. Admin screens (01–07) use the relaxed budget per docs/frontend-design-principles.md. Agency screens (08, 09) obey the strict budget — no metrics, no version numbers in primary copy, version info lives inside a collapsed Advanced section.
  • Versioning already exists. agent_prompt_revisions is live. Rings add one ring column + one membership table.
  • This composes with Explore/Execute Mode. Rings are per-revision safety (platform authors shipping changes); Explore/Execute is per-run safety (operators trying things). Orthogonal, not competing.
  • Platform stance: docs/spec-context.md currently says rollout_model: commit_and_revert and staged_rollout: never_for_this_codebase_yet. This feature is queued for when that flips (first live agency client).
  • No spec yet. These mockups are the review surface. Spec is drafted after the UX locks.
