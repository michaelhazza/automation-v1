Phase 1 — Auto Knowledge Retrieval

Mockups for review

Four mockups demonstrating the new knowledge model: Files alongside Documents on the Knowledge page, a one-click promotion flow from Files to Documents, the refreshed per-agent Data Sources tab with smart loading indicators, and the Documents tab with scope and mode visibility. Brief: docs/auto-knowledge-retrieval-dev-brief.md.

Reading order: Mockup 1 first (where the new tab lives), then Mockup 2 (the promotion flow), then Mockup 4 (Documents tab refresh), then Mockup 3 (per-agent view of how it all comes together).
MOCKUP 1
Knowledge — Files tab (new)
knowledge-files-tab.html
Knowledge page with the new Files tab alongside Authored memory, Auto-memory, and Documents. Shows agent-produced artifacts with three-dots menu open showing the "Add to Knowledge" action.
MOCKUP 2
Add to Knowledge modal
add-to-knowledge-modal.html
The promotion flow: file becomes a Document. Title, content preview, scope (defaulted from context), apply-to (defaulted to all agents), advanced expander with Auto / Always available / Reference only.
MOCKUP 3
Agent edit — Data Sources tab
agent-data-sources.html
Per-agent view of all linked documents across all five tiers (org / sub-account / agent / recurring task). No eager/lazy controls. Mode chips (Auto / Always available / Reference only) and a relevance bar showing how often each document is actually loaded.
MOCKUP 4
Knowledge — Documents tab refreshed
knowledge-documents-tab.html
Documents tab with new tier and mode chips, source provenance (manually authored / from file / from auto-memory / uploaded), and "used by N agents" indicator. Org-pinned and from-file documents have left-border highlights.
