Session limits
Soft session cap
How long one operator session can run before it wraps up and hands off. Shorter sessions keep costs predictable.
minutes
Auto-extend grace
If the operator is mid-step when the soft cap hits, it gets this extra time to finish the current step before stopping. Hard stop at soft cap + grace.
minutes
Task limits
Max chain length
A long task may run across multiple chained sessions. This caps how many sessions can chain before the task pauses for a human check-in.
sessions
Max wall-clock per task
A task that has been running (including pauses between sessions) for longer than this will pause for review, even if it has not used its session budget.
days
Per-task budget cap
Tasks pause for review when they use this many operator session minutes in total. About 50 sessions of 120 minutes at the default.
minutes
Concurrent operator sessions
How many operator runs can be active at the same time across this subaccount. Raising this lets more tasks run in parallel; lowering it protects compute budget.
max 25
Changes apply to new sessions only. In-progress sessions are not affected.
