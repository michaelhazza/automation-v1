TaskHeader: Auto-extend inline status
Component: client/src/components/openTask/TaskHeader.tsx
When the operator passes its soft session cap and is within the auto-extend grace period, TaskHeader temporarily shows an inline amber state. No modal is launched. The status row message replaces the normal pill. The "Stop" button stays visible so the operator can halt manually if needed. The amber tint applies only to the TaskHeader background, not to the panes below.
A — Past soft cap, auto-extending (22 of 30 min grace remaining)
Process May invoices
⌛ Finishing current step 22 min left to stop cleanly
The entire TaskHeader gets a light amber background (#fffbeb) with amber border. The normal "Operator running" pill is replaced with "Finishing current step" + a countdown chip showing grace minutes remaining. Pause button is hidden during auto-extend (task is winding down, pausing here would be confusing). Stop remains available for manual override.
B — Hard stop imminent (3 min remaining in grace)
Process May invoices
⌛ Finishing current step 3 min to hard stop
As the countdown nears zero, the chip shifts to an orange background (#fdba74) and the header background deepens slightly to #fff7ed. No modal, no toast, no audio. Inline only. If the operator does nothing, the session terminates naturally and the task transitions to the chain-handoff flow (or to Completed if the last session).

Design decision: The auto-extend state is not a modal. A modal would interrupt the operator and require a decision. Auto-extend is automatic — the system is handling it gracefully. The inline amber state communicates "something is happening, you can watch or intervene," not "you must decide now." This satisfies the brief's requirement: "subtle amber tinted background, NOT a modal."
