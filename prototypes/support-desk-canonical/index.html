Support Desk Canonical

Five clickable screens for the canonical support desk layer. Teamwork Desk as the first provider. Click any tile to open the screen.

All navigation between screens is live. Filters, mode selection, approve/reject buttons, and the inline edit flow on draft review all update visible state. Data is static and representative.

Deliberately not built

Per frontend design principles: primary task first, default to hidden.

  • Ticket volume dashboard with KPI tiles (tickets per day, first-response time, bot-vs-human ratio)
  • Raw internal IDs on user-facing rows (draft ID, agent run ID, idempotency key) — draft ID appears only on hover in the thread view
  • Provider status mapping editor (admin-only concern, separate surface)
  • Cross-inbox aggregate analytics and trend charts
  • Attachment inspector or attachment mirror management
  • Foundry training data export UI (lives in Foundry, not here)
