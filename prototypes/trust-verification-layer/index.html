Trust Verification Layer

Trust Verification Layer Prototype

Six screens extending the post-consolidation IA with three trust layers: skill verification (Layer 1), agent scorecards and model bench (Layer 2), and correction-sourced auto-memory (Layer 3). No new top-level navigation. All surfaces extend existing pages.

Date: 2026-05-07
Status: Round 1, awaiting operator review
Base prototypes: Consolidation 2026-05-06

Round 1 design decisions

  • model-bench is a focused page (not a modal): bench has a results + approval step that benefits from full-width table layout. Modal would be cramped.
  • Scorecard config is a new tab on agent-edit (not a section), consistent with the tab-per-domain pattern already established (Skills, Data sources, Budget, etc.).
  • Source filter added only to Auto-memory tab on Knowledge. Authored memory tab is manual-only by definition so the filter is redundant there.
  • home.html trust tile skipped: the existing 2x3 widget grid would be disrupted by a third column or an extra row. Brief permits this skip when the tile would crowd the design. Entry through Inbox + Govern / Quality is sufficient.
  • Verify summary bar added above the event list on run-trace as a quick scan strip (pass / fail / inconclusive counts). Adds no vertical height to individual rows.
  • Govern / Quality sidebar entry is a stub in the copied _sidebar.js; the operator reaches it from the topbar breadcrumb or direct link in this prototype.
Layer 1: Skill verification Verify badges on every run step
Run trace (extended)

Adds two trust affordances to the existing run trace layout: Verify badge per step (pass / fail / inconclusive / n/a) with hover tooltip showing the reason. Correct action on LLM-generated outputs (hover menu on llm_call and tool_result rows), opening a modal: original output (read-only), edited output, optional reason. Layer 2: Agent scorecards + model bench Quality tracking and model comparison
Agent edit: Scorecard tab (extended)
agent-edit-scorecard.html
New Scorecard tab on the agent edit page. Contains: scorecard dimensions list (name, description, floor threshold 0-1, enabled toggle, Add dimension button), sampling rate slider (default 20%), judge model selector, Run bench link, and a recent score summary strip (last 50 runs, mean per dimension, inline sparklines, drift warning).
Layer 2 Base prototype
Govern / Quality (new page)
govern-quality.html
New fourth Govern primitive. Sortable table of agents with: attached scorecard, 7-day mean score, 7-day sparkline trend, scored / total runs (sampling %), drift indicator (arrow + delta), last bench date. Filters: scorecard attached, drift state. Row click opens drawer with 30-day per-dimension trend chart, recent low-scoring runs (deep-link to trace), and Run bench action. Empty state when no scorecards exist.
New page Layer 2
Model bench (new page)
model-bench.html
Focused page (not a modal) for the bench workflow. Three states navigable via tabs at top: Setup (candidate model chips, sample count, scorecard preview, Run bench button), Running (progress bars per model, live mean score, cost so far, Cancel), Results (table with mean score, variance, latency, cost-per-task, composite winner highlighted, Approve as default per row).
