Trust Verification Layer Prototype

Ten screens extending the post-consolidation IA with three trust layers: skill verification (Layer 1), agent scorecards and model bench (Layer 2), and correction-sourced auto-memory (Layer 3). No new top-level navigation. All surfaces extend existing pages or add focused creation flows.

Date: 2026-05-08
Status: Round 2, awaiting operator review
Base prototypes: Consolidation 2026-05-06

Round 2 design decisions

  • "Override pass marks for this agent" deferred to Round 3. Multi-level nesting (collapsed section inside collapsible card inside tab) violates complexity budget. Will ship as an "Advanced" disclosure per attached scorecard once base multi-attach flow is validated.
  • scorecard-library.html renders as a standalone page with the same tab strip as govern-quality.html. The govern-quality.html Scorecards tab shows an iframe for prototype navigation, reflecting how the tab would work in-product.
  • Bench history tab in govern-quality.html shows approved/overridden/rejected outcomes. "Overridden" means the operator selected a different model than the recommended one — a real-world scenario worth showing.
  • agent-create.html uses two path tabs (System Agent / Template) on one page to make the inheritance story concrete without requiring the operator to navigate between two separate screens.
  • skill-create.html: "Advanced" code block defaults collapsed. The plain-English verify check is the primary surface; code is progressive disclosure for technical users.
  • govern-quality.html Scorecards tab: clicking opens scorecard-library.html as its own page (not an iframe). For the prototype, an iframe preview is also included in the Scorecards tab panel for convenience.
Layer 1: Skill verification Verify badges on every run step + custom skill creation flow
Run trace (extended)
run-trace.html
Verify badge per step (pass / fail / inconclusive / n/a) with hover tooltip showing the reason. Correct action on LLM-generated outputs: opens modal with original output (read-only), edited output, optional reason. After save, shows a "captured as auto-memory" toast.

Layer 2: Agent scorecards + model bench Quality tracking, scorecard library, bench, and creation flows
Agent create (new)
agent-create.html
Shows scorecard pre-attachment at agent creation. Two paths via tabs: Install a System Agent (Content Director) and Create from Template (Content Writer). Template path: all recommended scorecards are checked by default; "Don't include any scorecards" option available. Evaluation settings on both paths.
