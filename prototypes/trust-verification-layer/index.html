Trust Verification Layer

Trust Verification Layer Prototype

Ten screens extending the post-consolidation IA with three trust layers: skill verification (Layer 1), agent scorecards and model bench (Layer 2), and correction-sourced auto-memory (Layer 3). No new top-level navigation. All surfaces extend existing pages or add focused creation flows.

Date: 2026-05-08
Status: Round 3, awaiting operator review
Base prototypes: Consolidation 2026-05-06
Round 3, 2026-05-08: Simplification pass

Round 3 design decisions

  • model-bench.html: auto-select-15 replaced with an explicit multi-select table of last 50 runs (checkbox per row, newest first, 15 pre-ticked). "Pick newest 15" and "Clear all" convenience buttons added. Textarea paste-in replaced with vertical prompt cards with label and remove button. Cost estimates now append "(charged to this account's token budget)".
  • skill-create.html: Step 1 restructured into sub-steps 1A (name + description + "Suggest details" primary action) and 1B (parameters, blast radius, reversible toggle) revealed only after "Suggest details" is clicked. Blast radius labels simplified: "Just this skill" / "Your account" / "External services". "Tenant" removed. Help text added to all technical fields.
  • agent-edit-scorecard.html: "How often to grade" range slider replaced with a six-button segmented control (Off / 20% / 40% / 60% / 80% / 100%). Default 20% selected. Live help text updates on selection. "Off" disables drift detection.
  • run-trace.html: Verify badge states reduced from 4 to 3: Pass, Fail, Pending. "Not applicable" badge removed from lifecycle events; those steps show no badge at all. Summary strip reflects 3 states. Correct modal adds a reassurance note: "Your correction will appear in Knowledge for review and will improve future runs of this agent."
  • scorecard-library.html: Source pill reduced from 3 values (System / Organisation / This subaccount) to 2 (Platform / Custom). Custom pills carry a tooltip with the actual creation scope. Filter chips updated: All / Platform / Custom. Empty state shown when Custom filter matches no rows, with action buttons.
  • agent-create.html: Locked "Always attached" scorecard rows are now expandable via a caret toggle, revealing read-only quality checks (name, description, pass mark). Locked / no-detach status unchanged.
  • scorecard-create.html: Each pass mark input now shows a reference note below it with the typical range for similar checks ("Similar checks: 76-92%") or "No reference data yet" for new check types.
  • govern-quality.html: Empty states added to Agents tab (shown when filter returns no rows) and Bench history tab (togglable demo state).
  • "Override pass marks per agent" remains deferred. Still violates complexity budget at current UI depth.

Round 2 design decisions (archived)

  • "Override pass marks for this agent" deferred to Round 3 (still deferred, see above).
  • scorecard-library.html renders as a standalone page with the same tab strip as govern-quality.html. The govern-quality.html Scorecards tab shows an iframe for prototype navigation, reflecting how the tab would work in-product.
  • Bench history tab in govern-quality.html shows approved/overridden/rejected outcomes. "Overridden" means the operator selected a different model than the recommended one.
  • agent-create.html uses two path tabs (System Agent / Template) on one page to make the inheritance story concrete without requiring the operator to navigate between two separate screens.
  • skill-create.html: "Advanced" code block defaults collapsed. The plain-English verify check is the primary surface; code is progressive disclosure for technical users.
