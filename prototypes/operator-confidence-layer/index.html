Open-decision positions (5 of 5, Round 3 refinements noted)

1
Preview entry point
Compact two-option inline choice row inside NewBriefModal.tsx, below the schedule section (or below Priority for one-off tasks): "Create and run" (default) / "Preview first" (opt-in). Previous Round 1 position used a three-option segmented control on a separate page; Round 2 collapses to two options (no "Schedule only" option needed in the modal context) and grounds it on the actual modal surface.
2
Preview result format
Retained from Round 1: ordered step list (numbered, one action per step) below a two-sentence plain-English summary card. Each step has a skill chip and a "read only" green badge on non-mutating calls. No diff, no cost, no guarantee language. Round 2 adds a recurring context pill under the task name when the preview is for a recurring task.
3
Promote-to-run interaction
Retained from Round 1: single "Create recurring task" button on the Preview result page. Lands on ScheduledTaskDetailPage.tsx with a dismissable green success banner and a dismissable "Created from preview" lineage pill on the title row.
4
Undo affordance placement
Retained from Round 1: inline in the record's activity feed as an entry on the agent-edit event, plus a secondary header button when a snapshot is available. Button is disabled (not absent) when a human edited after the agent, with a tooltip explaining why.
5
Snapshot quota visibility
Updated Round 3: new "Storage" tab on Govern > Spending (SpendingPage.tsx). Single quota bar with per-agent breakdown table (informational only, no action buttons). Retention is automated at 30 days; no manual controls shown. Warning chip at 90%+ uses passive language ("auto-removed when quota is exceeded, no action needed"). No dedicated page; no KPI tiles; no manual retention selector.
