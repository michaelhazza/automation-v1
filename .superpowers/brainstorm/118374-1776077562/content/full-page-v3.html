Audit SEO
Performs a comprehensive on-page SEO audit of a website or specific page. Analyses technical SEO factors, meta information, content quality, and keyword alignment to identify issues and prioritise improvements.
Parameters
• url (required) — The page or site URL to audit • depth — How many pages to crawl (default: 1) • keywords — Target keywords to check alignment against • include_competitors — Whether to benchmark against top 3 SERP results
Instructions
Fetch the target URL using fetch_url. Analyse the following dimensions: TECHNICAL SEO - Page load time (infer from response headers) - Mobile viewport meta tag present - HTTPS enforced - Canonical tag present and self-referencing - robots meta tag — check for noindex - Structured data (JSON-LD) present and valid - XML sitemap discoverable via /sitemap.xml ON-PAGE SEO - Title tag: length (50–60 chars), keyword placement, uniqueness - Meta description: length (150–160 chars), call to action, keyword - H1 tag: single, contains primary keyword - H2–H6 structure: logical hierarchy, keyword variation - Image alt text: descriptive, not keyword-stuffed - Internal links: minimum 3 contextual links to related pages - External links: at least 1 authority source cited CONTENT QUALITY - Word count appropriate for topic (minimum 800 for informational) - Reading level appropriate for audience - No thin content sections (< 100 words per main section) - Clear value proposition in first 100 words OUTPUT FORMAT Produce a prioritised issue list grouped by severity (Critical / Important / Minor) with specific fix recommendations for each issue. End with an Overall SEO Score (0–100) and a 30-day action plan.

seo-content-audit
Deep editorial content audit focused on E-E-A-T signals, topical authority, and content gap identification. Evaluates whether content demonstrates genuine expertise and satisfies search intent at depth.
Parameters
• url (required) — Page URL to audit • target_keyword — Primary keyword or topic • competitor_urls — Up to 3 competitor pages covering the same topic • audience — Intended reader (e.g. "B2B marketers", "first-time homebuyers")
Instructions
Fetch the page content using fetch_url. If competitor_urls provided, fetch those too for benchmarking. E-E-A-T SIGNALS - Author byline: named author with credentials visible - First-hand experience: does the content cite personal testing, data, or case studies? - Publication date and last-updated date present - External citations: peer-reviewed or authoritative sources linked TOPICAL DEPTH - Does the page cover the topic more thoroughly than top-ranking competitors? - Are common user questions (check People Also Ask) answered? - Are related subtopics addressed or linked? CONTENT GAPS - Identify 3–5 subtopics covered by competitors but missing from this page - Flag any outdated statistics (> 2 years old) - Note any claims made without supporting evidence READABILITY - Flesch reading ease score estimate (aim 60+ for general audiences) - Average sentence length (target < 20 words) - Paragraph length (target < 4 sentences) KEYWORD ALIGNMENT - Primary keyword in title, first 100 words, at least one H2 - Semantic keyword variation (avoid exact-match stuffing) - Keyword density estimate (target 0.5–1.5%) OUTPUT FORMAT Score each dimension 0–100. Provide an overall Content Quality Score and a prioritised list of improvements with estimated impact (High / Medium / Low).
