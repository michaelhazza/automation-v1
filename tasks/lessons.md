# Agent Lessons Log

This file captures patterns from mistakes and corrections. Review it at the start of each session.
Add a new entry any time the user corrects a wrong assumption, bad output, or repeated error.

---

## How to Use This File

- After ANY correction: add an entry below in the format shown
- Be specific. Vague lessons do not prevent future mistakes.
- Group related lessons together
- If the same mistake appears twice, escalate it to the top of its section

## Entry Format

```
### [Date] - [Short description of the mistake]

**What happened:** [One sentence describing what went wrong]
**Root cause:** [Why it happened]
**Rule:** [The concrete rule to follow going forward]
```

### Example Entry

```
### 2025-03-01 - Marked task complete before testing

**What happened:** Told the user the feature was done without running it
**Root cause:** Assumed the logic was correct because it compiled
**Rule:** Never mark anything complete without running it and verifying output matches expected behaviour
```
