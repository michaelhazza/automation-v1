# iee-browser sandbox template

Playwright-backed IEE browser execution template for e2b sandboxes.

## Build status

⚠ **Not yet buildable today.** The CI sandbox-template-build pipeline is wired up after the e2b SDK is installed (see `SANDBOX-DEF-EGRESS-MECH` decision in `server/services/sandbox/e2bSandbox.ts`). Until then:

- `Dockerfile` carries a placeholder FROM digest and references `harness/dist/index.js` which is generated by the CI build step.
- `harness/package.json` + lockfile do not yet exist; CI adds them when bundling the executor.
- `PUBLISHED_VERSION` carries an all-zero `image_digest`; `assertNotLatestTemplateVersion` rejects this placeholder so any accidental production deploy fails at e2b provider construction time, not silently at runtime.
- The harness source (`harness/index.ts`) writes `status:'failed'` until the real Playwright executor is bundled — never silent success.

The sandbox provider only constructs in dev with `E2B_SDK_STUBBED=true`. The stub throws on every SDK call so this template never actually runs end-to-end today.

## Overview

Runs the IEE browser executor inside an e2b sandbox. The harness:
1. Reads the task payload from `/workspace/input.json`
2. Mounts the browser profile volume at `profileMount.userDataDirInSandbox` (default: `/workspace/profile`)
3. Executes the browser actions using Playwright
4. Writes output to `/workspace/output.json` and artefacts to `/workspace/artefacts`

## Version pinning

`CURRENT_VERSION` and `PUBLISHED_VERSION` are managed by the CI sandbox-template-build pipeline.
Update `base_image_digest` and `deps_lockfile_hash` in `CURRENT_VERSION` when bumping dependencies.

## Template name

`iee-browser` — referenced by `server/services/sandbox/e2bSandbox.ts` when `sandboxRequirement === 'browser'`.
