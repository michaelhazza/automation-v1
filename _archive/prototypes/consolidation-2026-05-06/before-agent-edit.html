SkillStudioPage and SkillAnalyzerPage are admin-only tools accessible separately from agent management. Operators configuring an agent must navigate: AdminAgentEditPage > Capabilities tab > edit link > AdminSkillEditPage. SkillStudio adds a third context-switch for custom skill creation. In the consolidated view, all skill management lives inside the agent's Skills tab with search-driven discovery scaling to 200+ skills.

Problems with the current state (Capabilities tab IA)

  • Single Capabilities tab cannot scale beyond ~20 skills before the operator must scroll endlessly to find a specific skill.
  • Data sources cramped at the bottom of the skill list — operators must scroll past all skill rows to reach them, and they are often missed entirely.
  • No search across the skill library. With 200+ skills available, finding a specific one requires manual scrolling through an unsorted flat list.
  • Tier inheritance (System / Org / This client) not visible per-row. Operators cannot tell which skills are platform-built, org-defined, or workspace-specific.
  • "Capabilities" is jargon. Operators do not know what the tab groups until they have already opened it. "Skills" and "Data sources" are self-describing.
  • No live filter. If 200+ skills are available, finding one is a scroll-marathon with no way to narrow by tier, category, or enabled state.
