Problems with the current state
  • Goals tree is maintained by operators but never consumed by agents. The subsystem has zero runtime integration.
  • Project description is free-form and not signposted as the agent-focusing field. Most operators use it for general notes, not focused objectives.
  • Setting projects.goalId gives operators the impression agents know about the goal. They do not. The link is metadata only.
  • OKR hierarchy (Mission / Objective / Key Result) adds navigational complexity for a feature that provides no agent benefit.
  • No clear separation between "what this project is trying to achieve" (per-project objective) vs "how the company strategises overall" (org Memory blocks, already implemented).
  • Agent context for tasks under a project is diluted: description may contain mix of objective, notes, and instructions. No structured field that agents reliably receive.
