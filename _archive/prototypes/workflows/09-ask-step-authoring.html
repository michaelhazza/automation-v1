State A · Default. The author lands here when they pick Ask as the step type. Two fields are already configured: audience (text, required) and tone (select, required, with three options). Below the field list, the small help text "Options: Professional warm / Friendly / Direct" is a peek at the select options without expanding the editor. Both dropdowns at the bottom are closed and showing their current value.
State B · Who can submit dropdown open. Four routing options. Three of them require additional config (Specific people = pick users; A team = pick a team; Org admin = no extra config) — the next click on any of them would surface the relevant picker inline. Task requester is the default and the most common pick because the requester is already on the task page when the form lands. The four-way routing is the same as Approval steps.
State C · Auto-fill dropdown open. V1 ships with two real options ("Don't auto-fill" and "Last completed run") plus two clearly-deferred V2 placeholders. Last-completed-run is the default because it's the value the operator gets the most leverage from with zero cost (skips silently on first run). The two V2 options are listed so the deferral is auditable, not invisible.
State D · Add a field type picker. Seven V1 field types in a tile picker. Standard form-renderer territory: text, textarea, select, multi-select, number, date, checkbox. The footer line names the deferrals so the author isn't left wondering "why no file upload?" — answer: V2. Picking any tile opens the field config inline (next state).
State E · Editing a select field. The audience field expanded for editing. Label (operator-visible), field key (binding identifier, auto-derived, read-only), help text, options list with drag-handles for reordering, required toggle. Same shape applies to all other field types except select-only fields drop the options list. Click "Done" to collapse back to a row.

The schema for everything in this inspector lives inside the existing params JSON column on the workflow-template-step row. No new tables. The form renderer that consumes this schema at runtime is a single component — see mock 10 for the runtime rendering of an authored Ask form.
