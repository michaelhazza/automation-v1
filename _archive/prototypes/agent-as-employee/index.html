What we did NOT build

Per CLAUDE.md frontend rules — primary task first, default to hidden, one primary action per screen — these surfaces are deliberately absent at launch:

  • Agent activity dashboard with KPI tiles
  • Agent productivity score / "tasks completed this week"
  • Timeline-style visualisation of every action — the audit log is tabular
  • Internal IDs (actor, identity, connector) on user-facing screens — diagnostic only
  • Documents surface (deferred to a follow-up; on Google, agents own files in Drive directly)
  • Standalone aggregate "Agent inbox" landing page across all agents in an org

If a customer asks for any of these, we add them as targeted single-task surfaces in a follow-up. Default is they don't exist.
