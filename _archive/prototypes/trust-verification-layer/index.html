Trust Verification Layer

Trust Verification Layer Prototype

Ten screens extending the post-consolidation IA with three trust layers: skill verification (Layer 1), agent scorecards and model bench (Layer 2), and correction-sourced auto-memory (Layer 3). No new top-level navigation. All surfaces extend existing pages or add focused creation flows.

Round 6, 2026-05-08: Terminology lock + slider bug fix

Round 6 design decisions

  • Terminology lock: "verify check" renamed to "runtime check" in all operator-facing copy across all screens. Schema field verify retained as developer literal. Umbrella name "Trust & Verification Layer" unchanged.
  • Slider bug resolved: agent-create.html System Agent and Template paths now correctly use the same 4-button quartile control (Off / 25% / 50% / 75%) as agent-edit-scorecard. The setGradeFreq() JS function that the buttons depended on was missing from agent-create.html; it is now added, with independent hint-text update per control.

Round 5 design decisions (archived)

  • run-trace.html: Correct dialog now exposes scope, persistence, and confidence so corrections feel auditable, not magic. "About this correction" block shows: Scope (This agent only), Persistence (Active on next run), Confidence (High signal — applied immediately, listed under Knowledge where you can edit, override, or reject).
  • model-bench.html: Bench results now include a Regression risk column (Low / Medium / High) derived from score variance. Addresses enterprise concern about hidden output degradation. Low = winner (Sonnet 4.6, +/-4%), Medium = Opus 4.7 (+/-3%), High = Haiku 4.5 (+/-9%, also fails pass mark).
  • Terminology rename ("verify" overloaded) deferred to spec phase per brief directives. Applied in Round 6.
  • First-class Policy primitive explicitly out of scope (brief); future Stage 4 candidate.

Round 4 design decisions (archived)

  • model-bench.html: Paste-in section defaults to 1 prompt card (was 3). Reduces friction when the user wants to paste a single prompt. The "Add prompt" button adds more cards as needed.
  • agent-edit-scorecard.html: "How often to grade" simplified from 6 buttons (Off / 20% / 40% / 60% / 80% / 100%) to 4-button quartile control (Off / 25% / 50% / 75%). 100% removed: if every run needs grading, the right answer is a stricter scorecard, not full sampling. Default 25%. Help text updated per quartile.
  • agent-create.html: Source pills (Platform-suggested / Organisation-suggested) removed from Suggested scorecard rows. At sub-account scope the distinction is irrelevant. All suggestions appear uniformly as "Suggested."
  • agent-create.html: Third locked/required row added: "PII redaction check." Represents an org-mandatory scorecard. Rendered identically to existing required rows (lock icon, Required badge, expandable caret). Sub-account users see no distinction between system-mandatory and org-mandatory — both are just "Required." Footer count updated to 3 required + 3 suggested.

Round 3 design decisions (archived)

  • model-bench.html: auto-select-15 replaced with an explicit multi-select table of last 50 runs. "Pick newest 15" and "Clear all" convenience buttons. Textarea paste-in replaced with vertical prompt cards. Cost estimates now append "(charged to this account's token budget)".
  • skill-create.html: Step 1 restructured into sub-steps 1A and 1B. Blast radius labels simplified. Help text added to all technical fields.
  • agent-edit-scorecard.html: "How often to grade" range slider replaced with a segmented control. Live help text updates on selection.
  • run-trace.html: Runtime check badge states reduced from 4 to 3: Pass, Fail, Pending. Correct modal adds reassurance note.
  • scorecard-library.html: Source pill reduced to 2 values: Platform / Custom. Empty state added when Custom filter has no rows.
  • agent-create.html: Locked "Always attached" rows made expandable via caret toggle to reveal read-only quality checks.
  • scorecard-create.html: Pass mark inputs show reference note with typical range for similar checks.
  • govern-quality.html: Empty states added to Agents and Bench history tabs.
  • "Override pass marks per agent" remains deferred.

Round 2 design decisions (archived)

  • "Override pass marks for this agent" deferred to Round 3 (still deferred, see above).
  • scorecard-library.html renders as a standalone page with the same tab strip as govern-quality.html. The govern-quality.html Scorecards tab shows an iframe for prototype navigation, reflecting how the tab would work in-product.
  • Bench history tab in govern-quality.html shows approved/overridden/rejected outcomes. "Overridden" means the operator selected a different model than the recommended one.
  • agent-create.html uses two path tabs (System Agent / Template) on one page to make the inheritance story concrete without requiring the operator to navigate between two separate screens.
  • skill-create.html: "Advanced" code block defaults collapsed. The plain-English runtime check is the primary surface; code is progressive disclosure for technical users.
