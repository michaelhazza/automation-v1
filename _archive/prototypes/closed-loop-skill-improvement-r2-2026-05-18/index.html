Automation OS

Closed-loop skill improvement

Hi-fi prototype, Round 2. 7 screens across two user roles. Agents propose, operators approve.

Flow: A scorecard failure triggers a proposal (S5 run trace). The subaccount admin reviews it in the queue (S1) and opens the drawer (S3) to accept, edit, or reject. Accepted amendments appear in the skill detail stack (S4). The org admin sees the same queue across all workspaces (S2) and has access to freeze controls (S7). Skills that hit the 20-amendment cap enter a paused state visible in the queue and on the skill page (S6).
Review queue surfaces
1
Surface A: In-workspace queue (subaccount admin)
Extends ReviewQueuePage. Full priority ordering: incident rollback alert, conflict banner, review_required header card, high-blast proposals, occurrence-count grouping, stale-soon chip, and grouped low-blast batch with one-click "Accept all". Workspace mode.
Click the group batch header to expand low-blast proposals. Click Org in the view mode pill to navigate to Surface B.
2
Surface B: Org cross-workspace queue (org admin)
Extends ReviewQueuePage driven by ViewModeSwitcher (Org mode). Aggregates pending proposals across all 6 workspaces. Queue-health metrics panel at the top surfaces structural problems first (stale ratio, conflict count, skills at cap). Governance overload alert triggers freeze-controls link. Same priority ordering as Surface A, with workspace column per row.
Workspace column on each row. Health panel shows 4 org-wide metrics.
Amendment review
3
Amendment review drawer
Opens from either queue surface. Shows kind pill, blast-radius badge, editable amendment text with char counter, diff view, failure trigger (red block), root-cause record (failure_mode tag and contributing factors), peer-review verdict, and collapsed provenance chain. Reject opens a categorical one-click panel (7 reject_reason enum values). Accept swaps to "Edit and accept" when text is changed.
Type in the text area to reveal "Edit and accept". Click Reject for the categorical reason selector. Expand "Provenance chain" and "Audit detail" for full traceability.
Skill detail
4
Skill detail with active amendment stack
Extends SubaccountSkillsPage and HistoryRender. Shows system base, 6 per-skill stack-health metrics (density, conflict rate, rollback rate, stale ratio, edit frequency, size trend), one retirement suggestion card for an inactivity-decay candidate, amendments in composition order with numeric position badges, lineage chain (collapsible) for one amendment showing a v1-to-v2 edit history, and collapsed retired amendments section.
Click "Lineage" on amendment #2 to see the lineage graph. Click the cap counter to demo amber and red states.
Run trace
5
Run trace: "Improvement proposed" event + composition tab
Extends RunTracePage and RunTraceEventRenderer. Scorecard-fail event (red) directly above the improvement-proposed event (violet) with kind pill, one-line summary, and "Review in queue" link. Below the event stream: a "Skill composition" panel with two tabs: Snapshot (resolver version, composed size, no truncation) and "Amendments used" (which amendments were included in this run and which were excluded, with reasons).
Switch between "Snapshot" and "Amendments used" tabs in the Skill composition panel.
Lifecycle states
6
Review-required warning surface (cap reached)
Extends SubaccountSkillsPage. The "Paused" badge appears in the page header when the 20-amendment cap is hit. A warning banner explains what is suppressed, why, and the exact steps required to thaw proposal generation. Suppressed proposals table shows what was generated but not surfaced. Active stack section leads with retirement suggestions and a "show more candidates" expander. Directive but not alarmist per trust-posture guidelines.
Click "Retire" on any suggestion to see the slot-freed toast. "Pause proposal generation" links to S7.
7
Governance freeze controls (org admin)
Admin-only surface. Scope selector (org / per workspace / single skill). Freeze matrix: 4 independently composable layers (proposal generation, proposal surfacing, amendment activation, replay execution) each described in plain English. Existing accepted amendments are unaffected by any freeze. Toggling any layer requires a typed reason before applying (audit requirement). Freeze event log shows who froze what, when, and for how long. Framed as a circuit breaker.
Toggle any freeze switch and scroll down to see the reason field and confirm. Acme Corp shows generation already frozen as a demo state.
Design decisions, Round 2
  • S1 extended with full §4.4 priority ordering (incident, conflict, review_required, high blast, high occurrence, stale-soon, grouped low). Round 1 had a flat list.
  • Surface B (S2) is a separate dedicated screen (not a toggle on S1) to keep each surface's information density appropriate to its role.
  • S3 drawer now includes: reject reason enum (7 categorical options), root-cause record block, collapsed provenance chain, and collapsed full audit detail. Round 1 had basic "Why proposed" only.
  • S4 skill detail adds: 6 stack-health metrics, retirement suggestion card with inactivity reasoning, composition-order numbered badges, and lineage graph for one amendment.
  • S5 run trace adds: "Skill composition" panel with Snapshot and "Amendments used" tabs covering §3.8 composition observability.
  • S6 (review_required) and S7 (governance freeze) are new screens not present in Round 1.
  • No em-dashes in any copy. "Paused" not "Blocked". "Apply" not "Approve". "Why this was proposed" not "Why this is correct". "Circuit breaker" not "kill switch".
  • Admin-only controls (S7 governance freeze, org-admin pill) absent from non-admin views per design rules.
